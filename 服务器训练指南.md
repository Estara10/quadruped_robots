# 服务器训练指南

## 核心概念

**训练跑在服务器上，不跑在你电脑上。** 你的电脑只是一个"遥控器"——通过 SSH 给服务器发指令，服务器执行。发完指令后，关机、断网、合上笔记本，服务器继续跑，不受影响。

类比：用手机 App 遥控扫地机器人开始清扫，然后手机关机，扫地机器人还在扫。

### 服务器配置

| 对比 | 笔记本 | 服务器 |
|------|--------|--------|
| GPU | RTX 4060（8GB） | 4× A800（每张 80GB） |
| 内存 | 16-32GB | 503GB |
| CPU | 笔记本 CPU | 32 核 |
| 用途 | 日常使用 | 专门跑训练 |

### tmux 是什么

正常 SSH：终端里启动训练，关闭终端 → 训练停。

**tmux 解决的就是这个问题**：创建一个独立于终端的会话，训练程序挂在 tmux 里运行，和你的终端窗口解耦。关终端、断网、关机，tmux 在服务器上继续跑。

## 每日连接流程

每天打开电脑后的标准操作：

### 1. 打开终端，连上服务器

```bash
ssh ABS_training
```

输入密码。

### 2. 看看有哪些 tmux 会话在跑

```bash
tmux ls
```

会列出类似这样的输出：

```
abs_agile: 1 windows (created Mon May 11 15:30:00 2026)
abs_tensorboard: 1 windows (created Mon May 11 15:35:00 2026)
```

如果没有会话（报错 `no server running`），说明上次没启动训练，或者服务器重启过。

### 3. 进入训练会话

```bash
tmux attach -t abs_agile
```

现在你看到训练正在跑的界面了。可以观察最新日志：
- `iter XXXX/4000` — 当前迭代数
- `mean_reward` — 奖励值（应该逐步上升）
- `value_loss`, `surrogate_loss` — PPO 损失值

### 4. 检查 GPU 占用情况

在 tmux 里（或另开终端），查看 GPU 是否有人在用：

```bash
nvidia-smi
```

重点关注 GPU 1（你预约的）：
- 显存占用：程序在用就是正常的
- GPU 利用率：80%-100% 说明在跑，0% 说明出问题了

### 5. 离开

看完就可以了，训练继续跑：

- **`Ctrl+B` `D`** 断开 tmux（训练不中断）
- `exit` 断开 SSH
- 关终端、关机都可以

## 首次启动训练

第一次训练时用这个流程（和每日连接的流程不同——每日连接是 attach 已有会话，这里要创建新会话）。

### 第一步：连接服务器

```bash
ssh ABS_training
```

### 第二步：创建 tmux 会话

```bash
tmux new -s abs_agile
```

tmux 内底部有绿色状态栏，表示你在 tmux 里。

### 第三步：激活环境，启动训练

```bash
conda activate abs
cd /data/sxq/ABS/training/legged_gym/legged_gym
CUDA_VISIBLE_DEVICES=1 python scripts/train.py --task=go1_pos_rough --num_envs=1280 --max_iterations=4000
```

看到训练日志输出（`iter 0/4000`, loss 数值……），说明跑起来了。

### 第四步：断开 tmux（关键）

按 **`Ctrl+B`，松开，再按 `D`**（detach）。

回到普通终端，训练继续跑。

> 绝对不要用 `exit` 或 `Ctrl+C`！那会杀掉训练。

### 第五步：断开服务器

```bash
exit
```

此时可以关机，训练在服务器上继续。

## tmux 常用命令

### 会话管理

| 命令 | 作用 |
|------|------|
| `tmux new -s <名字>` | 创建新会话 |
| `tmux ls` | 列出所有会话 |
| `tmux attach -t <名字>` | 进入已有会话 |
| `tmux kill-session -t <名字>` | 删除会话（训练也会停） |
| `tmux rename-session -t <旧名> <新名>` | 重命名会话 |

### 窗口操作（tmux 内部使用）

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+B` `C` | 创建新窗口 |
| `Ctrl+B` `N` | 切换到下一个窗口 |
| `Ctrl+B` `P` | 切换到上一个窗口 |
| `Ctrl+B` `,` | 重命名当前窗口 |
| `Ctrl+B` `W` | 列出所有窗口（可用方向键选择） |
| `Ctrl+B` `&` | 关闭当前窗口 |

### 分屏操作（tmux 内部使用）

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+B` `%` | 垂直分屏（左右两个） |
| `Ctrl+B` `"` | 水平分屏（上下两个） |
| `Ctrl+B` 方向键 | 切换到相邻窗格 |
| `Ctrl+B` `X` | 关闭当前窗格 |

### 滚动查看历史输出

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+B` `[` | 进入滚动模式（可用方向键/PgUp/PgDn 翻页） |
| `q` | 退出滚动模式 |

## 同时跑多个训练

可以在一个 tmux 会话里创建多个窗口：

```bash
tmux new -s abs_train              # 创建主会话
conda activate abs
cd /data/sxq/ABS/training/legged_gym/legged_gym
CUDA_VISIBLE_DEVICES=1 python scripts/train.py --task=go1_pos_rough --num_envs=1280

# 按 Ctrl+B C 创建新窗口
# 在新窗口里跑别的，比如 tensorboard
tensorboard --logdir /data/sxq/ABS/training/legged_gym/logs --port 6006 --bind_all
```

## 注意事项

1. **只用 GPU 1**（已预约）：`CUDA_VISIBLE_DEVICES=1`
2. **不修改 `/data/isaacgym/`**（共享安装）
3. **不 kill 任何进程**（别人可能在用）
4. **不修改系统级配置**
5. 训练中断/想停：进入 tmux 会话后按 `Ctrl+C` 终止
6. **添加新机器人配置后必须重装 legged_gym**，否则新文件不会被识别：

```bash
cd /data/sxq/ABS/training/legged_gym
pip install -e . -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 训练模块完整流程

### 1. Agile Policy
```bash
CUDA_VISIBLE_DEVICES=1 python scripts/train.py --task=go2_pos_rough --num_envs=1280 --max_iterations=4000 --headless
```

### 2. 导出 Agile Policy
```bash
CUDA_VISIBLE_DEVICES=1 python scripts/play.py --task=go2_pos_rough --num_envs=1
# 看到 "Exported policy as jit script to:" 后 Ctrl+C
```

### 3. RA Value Network
```bash
CUDA_VISIBLE_DEVICES=1 python scripts/testbed.py --task=go2_pos_rough --headless --trainRA --num_envs=1280
```

### 4. Recovery Policy
```bash
CUDA_VISIBLE_DEVICES=1 python scripts/train.py --task=go2_rec_rough --num_envs=1280 --max_iterations=6000 --headless
```

### 5. 导出 Recovery Policy
```bash
CUDA_VISIBLE_DEVICES=1 python scripts/play.py --task=go2_rec_rough --num_envs=1
# 看到 "Exported policy as jit script to:" 后 Ctrl+C

# 复制到 testbed 期望的位置
cp /data/sxq/ABS/training/legged_gym/logs/go2_rec_rough/exported/policies/<文件名>.pt \
   /data/sxq/ABS/training/legged_gym/resources/policy/recover_v4_twist.pt
```

### 6. 端到端测试
```bash
CUDA_VISIBLE_DEVICES=1 python scripts/testbed.py --task=go2_pos_rough --headless --num_envs=1000 --testRA
```
